# Documentation Overview

`docs/` centralizes the knowledge that API consumers, developers, and operators need around `searchService`. Use the matrix below to jump to the right reference:

| Document | Purpose | Primary Audience |
| --- | --- | --- |
| `docs/search-service-api.md` | API contract, integration flow, and usage examples for services calling the hybrid search API. | Upstream service developers |
| `docs/ci-cd.md` | Local verification steps, automated test matrix, CI workflow, and load/perf guidance. | Contributors / reviewers |
| `docs/operational-readiness.md` | Logging, monitoring, SLOs, alert rules, deployment gates, and incident response. | SRE / platform / on-call |
| `docs/security.md` | Security scanning, dependency vulnerability policy, and remediation expectations. | Security champions / leads |
| `docs/repository-structure.md` | Repository layout, ownership guardrails, and expectations for new components. | All developers |

When adding a new document, update this table so future readers know where to start.
